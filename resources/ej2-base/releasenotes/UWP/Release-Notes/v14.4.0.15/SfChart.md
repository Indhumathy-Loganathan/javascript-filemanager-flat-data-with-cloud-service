## SfChart

### Bug Fixes
{:#sfchart-bug-fixes}


*  \#163515 - Now, [`PieSeries`](https://help.syncfusion.com/uwp/sfchart/series#pie-and-doughnut-charts) will update properly while dynamically changing its data points.

*  \#163834 - Now, [`FastColumnBitmapSeries`](https://help.syncfusion.com/uwp/sfchart/series#fast-column) range will be set properly while providing the negative data points.

*  \#166167 - Now CPU will not increase when updating existing data points.

*  \#166716 - [`CircularSeries`](https://help.syncfusion.com/uwp/sfchart/series#pie-and-doughnut-charts) with [`smart labels`](https://help.syncfusion.com/uwp/sfchart/adornments#smart-labels-for-accumulation-series) will no longer hang while changing the [`Legend's`](https://help.syncfusion.com/uwp/sfchart/legend) position.

*  \#166744 - [`ZoomPosition`](https://help.syncfusion.com/cr/cref_files/uwp/Syncfusion.SfChart.UWP~Syncfusion.UI.Xaml.Charts.ChartAxisBase2D~ZoomPosition.html) of [`SfChart`](https://help.syncfusion.com/uwp/sfchart/getting-started) and [`SfDateTimeRangeNavigator`](https://help.syncfusion.com/uwp/sfdatetimerangenavigator/getting-started) will no longer reset while adding data dynamically.


### Features
{:#sfchart-features}


*  Provided [`ChartAdornment`](https://help.syncfusion.com/cr/cref_files/uwp/Syncfusion.SfChart.UWP~Syncfusion.UI.Xaml.Charts.ChartAdornment.html) as DataContext for [`ChartAdornmentInfo's`](https://help.syncfusion.com/cr/cref_files/uwp/Syncfusion.SfChart.UWP~Syncfusion.UI.Xaml.Charts.ChartAdornmentInfo.html) [`SymbolTemplate`](https://help.syncfusion.com/cr/cref_files/uwp/Syncfusion.SfChart.UWP~Syncfusion.UI.Xaml.Charts.ChartAdornmentInfoBase~SymbolTemplate.html).

*  Provided Z-Axis support for [`SfChart3D`](https://help.syncfusion.com/uwp/sfchart/3d-charts).
