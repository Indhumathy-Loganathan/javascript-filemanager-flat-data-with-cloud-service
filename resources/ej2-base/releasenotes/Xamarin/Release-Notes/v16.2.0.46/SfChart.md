## SfChart

### Bug Fixes
{:#sfchart-bug-fixes}

* \#205789 – The `InvalidArgumentException` will no longer be thrown when double tapping the range series in chart.
 
* \#208922 – Custom colors will be applied properly in UWP platform.
 
* \#209985 – The legend items label will be updated at run time in UWP platform.
 
* \#207204, 208178 – The issue with horizontal panning stops working after panning to the leftmost edge has been fixed in iOS platform.
 
* \#209617 – The labels of [`CategoryAxis`](http://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.CategoryAxis.html) are rendering properly when the [`EnableAutoIntervalOnZooming`](http://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.ChartAxis~EnableAutoIntervalOnZooming.html) property of `ChartAxis` is set to true.
 
* \#206590 – The issue with improper position of axis labels when the [`LabelsIntersectionAction`](http://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.ChartAxis~LabelsIntersectAction.html) property is set to `MultipleRows` in iOS platform has been fixed.
 
* \#209638 – The `IllegalArgumentException` will no longer be thrown when zooming column series with gradient color in Android 8.0 version.
 
* \#196661 – [`ViewAnnotation`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.ViewAnnotation.html) will be clipped properly when zooming the chart with coordinate axis.
