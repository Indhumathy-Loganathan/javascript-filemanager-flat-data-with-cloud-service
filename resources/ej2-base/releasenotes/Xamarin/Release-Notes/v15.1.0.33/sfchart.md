## SfChart

### Improvement
{:#sfchart-improvement}

* \#169601- Support to specify hollow/solid candles in [`CandleSeries`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.CandleSeries.html).

### Bug fixes
{:#sfchart-bug-fixes}

* \#170154 - Tooltip will no longer throw exception of type 'Java.Lang.Error' while navigating between pages.
* \#170751 - Trackball labels will work properly while zooming.
* \#170706 - Trackball will render properly for the series which has empty data source.
* \#170040 -  Now, the [`ChartDataMarker`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.ChartDataMarker.html) is rendering properly when the animation is enabled for [`SplineSeries`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.SplineSeries.html).
* \#167292 - Now, the [`TrackballLabelTemplate`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.CartesianSeries~TrackballLabelTemplateProperty.html) property is working properly when configuring the [`SfChart`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.SfChart.html) in OnAppearing Method.
* \#169010 - Now the [`ChartSeries`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.ChartSeries.html) are not overlapped when toggling its visibility using [`ChartLegend`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.ChartLegend.html).
