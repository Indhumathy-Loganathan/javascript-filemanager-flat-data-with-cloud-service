## SfLinearGauge

### Bug fixes
{:#sflineargauge-bug-fixes}

* \# 137575 – The linear gauge [`Scale`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfGauge.XForms~Syncfusion.SfGauge.XForms.LinearScale.html) displays its maximum value according to the value specified in the [`MaximumValue`](https://help.syncfusion.com/cr/xamarin/Syncfusion.SfGauge.XForms~Syncfusion.SfGauge.XForms.LinearScale~MaximumValue.html) property.

* \# 210575 – – Now,[`BarPointer`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfGauge.XForms~Syncfusion.SfGauge.XForms.BarPointer.html) has rendered properly when applying small value for [`CornerRadius`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfGauge.XForms~Syncfusion.SfGauge.XForms.BarPointer~CornerRadius.html) in Xamarin.Forms for Android.

* \# 210575 – The [`GradientStops`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfGauge.XForms~Syncfusion.SfGauge.XForms.LinearScale~GradientStops.html) in [`LinearScale`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfGauge.XForms~Syncfusion.SfGauge.XForms.LinearScale.html) can now be updated dynamically in Xamarin.Forms for iOS and UWP.
