## SfLinearGauge

### Bug fixes
{:#sflineargauge-bug-fixes}

* \# 215230 - [UWP] [`SfLinearGauge`](https://help.syncfusion.com/xamarin/sflineargauge/overview) will be rendered with the available size.
* \# 224262, #226639 - [`Annotation`](https://help.syncfusion.com/xamarin/sflineargauge/annotations#annotation) view will be shown when adding custom annotation views inside the layouts.
* \# 225581 - The `ObjectDisposedException` will no longer be thrown when adding the linear gauge in `ListView`.
* \# 225513 - [iOS] [`LinearScale`](https://help.syncfusion.com/xamarin/sflineargauge/scales#linear-scale)'s [`Pointers`](https://help.syncfusion.com/xamarin/sflineargauge/pointers) collection will be updated properly when changing the pointer collection dynamically.
* \# 226627 - Gradient colors will be applied properly when changing the start and end values of the [`Range`](https://help.syncfusion.com/xamarin/sflineargauge/ranges#setting-start-and-end-values-for-range).
* \# 225513 – Labels will be visible when changing the [`ShowLabels`](https://help.syncfusion.com/xamarin/sflineargauge/labels#labels-visibility) value dynamically with multiple scales. 
* \# 143039 – [Android] The value of [`BarPointer`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfGauge.XForms~Syncfusion.SfGauge.XForms.BarPointer.html) will be updated properly when setting the decimal points.

### Features
{:#sflineargauge-features}

* \#221892 - Support has been provided to set the duration for [`Pointers`](https://help.syncfusion.com/xamarin/sflineargauge/pointers) Animation.
