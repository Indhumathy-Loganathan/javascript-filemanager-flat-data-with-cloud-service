## SfTextInputLayout

### Improvements
{:#sftextinputlayout-improvements}

* \#I223690 - Support has been extended to Android 4.4 (API 19) for [`SfTextInputLayout`](https://help.syncfusion.com/cr/xamarin/Syncfusion.Core.XForms~Syncfusion.XForms.TextInputLayout_namespace.html).

### Bug fixes
{:#sftextinputlayout-bug-fixes} 

* \#F141868 - [`Hint`](https://help.syncfusion.com/cr/xamarin/Syncfusion.Core.XForms~Syncfusion.XForms.TextInputLayout.SfTextInputLayout~HintProperty.html) and the placeholder text will now be displayed at their allocated positions at load time.
* \#F141748 - [`Hint`](https://help.syncfusion.com/cr/xamarin/Syncfusion.Core.XForms~Syncfusion.XForms.TextInputLayout.SfTextInputLayout~HintProperty.html) floats properly when removing and adding the same instance of the view in layouts.
* \#I223411,#I225464 - The control will be aligned properly when [`SfTextInputLayout`](https://help.syncfusion.com/cr/xamarin/Syncfusion.Core.XForms~Syncfusion.XForms.TextInputLayout_namespace.html) is placed inside [`FlexLayout`](https://docs.microsoft.com/en-us/xamarin/xamarin-forms/user-interface/layouts/flex-layout) with different types of layout hierarchy.
