## SfChart

### Bug Fixes

{:#sfchart-bug-fixes}

* \#185509 - Now, the axes labels will render properly even after zooming when Minimum and Maximum properties are set explicitly.

* \#185450 - Now, the data marker will render properly in [`PieSeries`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.PieSeries.html) while using smart labels. 

* \#185696 – Issue with truncating percentage in data marker label has been fixed in [`PieSeries`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.PieSeries.html).

* \#185542 -  Now, the data marker labels for [`RangeColumnSeries`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.RangeColumnSeries.html) will render properly.

* \#185660 -  The issue with [`ChartAxis`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.ChartAxis.html) ticks position has been fixed.

* \#184523 – Now, the exception “Can’t access disposed object” will no longer be thrown.

* \#185571, #185619 – Now, the [`ChartSeries.IsVisible`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.ChartSeries~IsVisible.html) property will change dynamically when the series visibility is changed using [`ChartLegend`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.ChartLegend.html).

* \#186152, #183740- [`StackingColumnSeries`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.StackingColumnSeries.html) will no longer throw System.InvalidCastException.

### Improvements

{:#sfchart-improvements}

* \#184837, #184921 - Improved the [`ScatterSeries`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.SfChart.XForms~Syncfusion.SfChart.XForms.ScatterSeries.html) performance while zooming and panning.
