## SfTextInputLayout

### Bug fixes
{:#sftextinputlayout-bug-fixes}

* \#F149298, F148648, F149723 - The [`Hint`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.Core.XForms~Syncfusion.XForms.TextInputLayout.SfTextInputLayout~Hint.html) label position and the [`SfTextInputLayout`](https://help.syncfusion.com/cr/xamarin/Syncfusion.Core.XForms~Syncfusion.XForms.TextInputLayout.SfTextInputLayout.html) height are updated properly while providing lengthy text in the ‘Editor’ at load time.

* [UWP] The ‘ArgumentOutOfRangeException’ will no longer be thrown by setting the [`InputViewPadding`](https://help.syncfusion.com/cr/cref_files/xamarin/Syncfusion.Core.XForms~Syncfusion.XForms.TextInputLayout.SfTextInputLayout~InputViewPadding.html) property by using the ‘Style’ property in both `Load time` and `Run time`.

### Features
{:#sftextinputlayout-features}

 * [Android, iOS] Added support for input views such as picker, date picker and time picker.
 * Added support for customizing leading and trailing views width and height (limited to 48 px for height).
