## SfChart

### Bug Fixes
{:#sfchart-bug-fixes}


*  \#159928 - Now, [`FastLineSeries`](https://help.syncfusion.com/wpf/sfchart/series#fast-charts) with non-linear data points will no longer disappear while [`zooming`](https://help.syncfusion.com/wpf/sfchart/interactive-features#zoom-and-pan).

*  \#159909 - Performance issue will no longer occur while binding the interior property to [`ChartSegment`](https://help.syncfusion.com/cr/cref_files/wpf/Syncfusion.SfChart.WPF~Syncfusion.UI.Xaml.Charts.ChartSegment.html).

*  \#161668 - Now, [`Scrolling`](https://help.syncfusion.com/wpf/sfchart/interactive-features#resizable-scrollbar) of the [`SplineSeries`](https://help.syncfusion.com/wpf/sfchart/series#line-and-spline-charts) with [`CustomTemplate`](https://help.syncfusion.com/cr/cref_files/wpf/Syncfusion.SfChart.WPF~Syncfusion.UI.Xaml.Charts.SplineSeries~CustomTemplate.html) will work properly.

*  \#161604, #161609 - In [`SelectionZooming`](https://help.syncfusion.com/wpf/sfchart/interactive-features#zoom-and-pan), selected region is setting proper while applying [`PlotOffset`](https://help.syncfusion.com/cr/cref_files/wpf/Syncfusion.SfChart.WPF~Syncfusion.UI.Xaml.Charts.ChartAxis~PlotOffset.html) or [`EdgeLabelsDrawingMode`](https://help.syncfusion.com/cr/cref_files/wpf/Syncfusion.SfChart.WPF~Syncfusion.UI.Xaml.Charts.ChartAxis~EdgeLabelsDrawingMode.html) as [`Fit`](https://help.syncfusion.com/cr/cref_files/wpf/Syncfusion.SfChart.WPF~Syncfusion.UI.Xaml.Charts.EdgeLabelsDrawingMode.html).
