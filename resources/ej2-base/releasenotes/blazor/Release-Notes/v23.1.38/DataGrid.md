## Data Grid

### Bug Fixes

- `#493861` - Resolved the issue, where the validation tooltip content was not aligning properly when the [cell](https://blazor.syncfusion.com/documentation/datagrid/cell) width was too small.
- `#I495952` - Resolved the issue, when using a non-nullable datetime column, the [filtering](https://blazor.syncfusion.com/documentation/datagrid/filtering) was not working properly.
- `#I497869` - Resolved the issue where the filter operator was not working properly while showing suggestions in the [SfAutoComplete](https://blazor.syncfusion.com/documentation/autocomplete/getting-started) component rendered inside the [Excel filter](https://blazor.syncfusion.com/documentation/datagrid/excel-like-filter).
- `#I492248` - Fixed the issue where the [footer aggregate](https://blazor.syncfusion.com/documentation/datagrid/aggregates#footer-aggregate) was not rendering correctly when using [adaptive layout](https://blazor.syncfusion.com/documentation/datagrid/adaptive-layout) in mobile mode and in [vertical row rendering](https://blazor.syncfusion.com/documentation/datagrid/adaptive-layout#vertical-mode) mode.
- `#I501345` - Resolved the issue where [initial grouping](https://blazor.syncfusion.com/documentation/datagrid/grouping#initial-group) did not work properly when the grid was bound with [column template](https://blazor.syncfusion.com/documentation/datagrid/column-template) and [column visibility](https://blazor.syncfusion.com/documentation/datagrid/columns#visibility).
- `#I497352` - Fixed the issue where the tooltip was not displayed properly when the grid was rendered inside the [SfDialog](https://blazor.syncfusion.com/documentation/dialog/getting-started).
- `#FB46848` - Resolved the issue where template columns were exported without considering the column [visibility](https://blazor.syncfusion.com/documentation/datagrid/columns#visibility) during [PDF Export](https://blazor.syncfusion.com/documentation/datagrid/pdf-export).
